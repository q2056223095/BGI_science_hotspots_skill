# 工作流检查清单

## A. 选题判断

- [ ] 这条热点属于发现型 / 机制型 / 技术型 / 争议反常识型哪一类？
- [ ] 它是否有明确的科学事实或原始来源？
- [ ] 它是否适合小红书表达？
- [ ] 它是否有容易被误读的地方？
- [ ] 它是否需要额外事实核验？

---

## B. 事实核验

- [ ] 是否核准研究机构 / 团队？
- [ ] 是否核准发布时间？
- [ ] 是否核准发表期刊或权威来源？
- [ ] 是否核准论文题目或新闻原文？
- [ ] 是否核准研究对象、样本数量或证据来源？
- [ ] 是否保留关键数字？
- [ ] 是否明确研究边界？
- [ ] 是否区分事实、推测和传播解读？

---

## C. 文案检查

- [ ] 是否不是单纯新闻复述？
- [ ] 开头是否有 3 秒吸引力？
- [ ] 是否把复杂科学讲成了大白话？
- [ ] 是否有具体事实、数字或真实场景？
- [ ] 是否讲清“为什么重要”？
- [ ] 是否保留科学边界？
- [ ] 是否避免强行转教育、课程或品牌？
- [ ] 结尾是否自然收束，而不是喊口号？

---

## D. 去 AI 味检查

- [ ] 是否减少机械排比？
- [ ] “不是……而是……”是否不超过 2 次？
- [ ] 是否避免连续堆叠抽象大词？
- [ ] 是否避免“这才是”“真正的”“未来一定”等口号表达？
- [ ] 是否读起来像真实编辑修改过，而不是一次性生成？
- [ ] 是否有克制的个人判断？

---

## E. 图片检查

- [ ] 是否为 5 张独立图，而不是拼成长图？
- [ ] 是否适合小红书 3:4？
- [ ] 每张图是否只有一个核心结论？
- [ ] 是否保持统一视觉风格？
- [ ] 是否有明显的信息层级？
- [ ] 是否参考了 `assets/` 中同类型视觉资产？
- [ ] 是否避免低幼风、电商风、医疗广告风、过度赛博风？
- [ ] 是否手机端可读？

---

## F. 合规检查

- [ ] 健康类内容是否避免医疗承诺？
- [ ] 是否避免把动物实验、体外实验或机制研究直接外推到人？
- [ ] 是否避免“已经证明”“一定有效”“彻底改变”等过满表达？
- [ ] 技术类内容是否区分预测、验证、实验和应用？
- [ ] 科研发现是否避免“彻底推翻”式夸大？
- [ ] 图片是否避免直接搬运论文图或机构图？
- [ ] 是否没有过强销售感或强行品牌转化？

---

## G. 输出交付

- [ ] 标题是否足够可选（至少 5 个）？
- [ ] 正文是否可直接发？
- [ ] AI 味自查评分是否达标？
- [ ] 图片规划是否明确？
- [ ] 图片提示词是否包含视觉参考约束？
- [ ] 如需出图，是否为 5 张独立成图？
- [ ] 是否保留必要科学边界提醒？
